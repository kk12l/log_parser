Парсер логов - тестовое

Запуск с помощью mojolicious - 'morbo myapp.pl'. Приложен файл для инициализации БД.
В корневом пути сайта ссылки на загрузку данных и поиск.

Судя по тексту задания, можно предположить, что его составлением занималось несколько человек.
Схема была адаптирована для MySQL.
В задании перепутаны ID_записи_в_БД и ID_в_логе. Я опёрся на структуру БД как источник правды - первичный ключ 'id' для БД, 'int_id' для данных из лога.
В таблице message нет поля 'address', значит адрес записываю в 'str'. Поиск организую по нему, что подразумевает попадание в результаты лишних данных на основе подстроки из лога. Показываемые найденные записи будут включать адрес, если эти записи ищ таблицы message.
